Section 14.2 — Overview

Multiple Access — The Big Picture

Wireless spectrum is scarce and expensive. Many users want to communicate simultaneously. How do we fairly and efficiently divide limited wireless resources? That is the question Multiple Access answers.

Step 1 — The Fundamental Problem

Imagine a cellular network. Hundreds of users simultaneously want to make calls, stream video, and send messages. But the wireless system has fixed, limited resources:

  • Limited bandwidth — only a certain amount of radio spectrum is available
  • Limited time — only so many transmissions possible per second
  • Limited signal space — the number of orthogonal channels is bounded
Highway analogy: If only one car uses a 10-lane highway, it's a waste. We divide it into lanes so many cars travel simultaneously. Multiple access divides wireless resources into "lanes" for different users.

Step 2 — What Multiple Access Does

Multiple Access = techniques that divide the total signal space into channels and assign these channels to different users. Users with continuous transmission needs (voice, video) get dedicated channels — guaranteed access that won't be interrupted.

The available signal dimensions can be divided along:

Time

→ TDMA
Users take turns in time slots

Frequency

→ FDMA
Users get different frequency bands

Code

→ CDMA
Users share time+freq but differ by code

Space

→ SDMA
Users in different directions get same freq/time

Step 3 — Orthogonal vs Non-Orthogonal Division

How cleanly are the channels separated?

MethodOrthogonalityInterference between users
TDMAOrthogonal (different time slots)Zero (if perfectly synchronised)
FDMAOrthogonal (different frequencies)Zero (with ideal filters)
CDMA (orthogonal codes)OrthogonalZero
CDMA (non-orthogonal codes)Non-orthogonalNonzero — users interfere

Step 4 — Signal Space Dimension: The Unifying Concept

📐
What is signal space dimension? In communication theory, the total number of orthogonal signals that can fit in a channel of bandwidth B over time T is:
Signal space dimension = 2BT B = bandwidth (Hz) · T = symbol duration (s) · This is the maximum number of independent signals (dimensions) available

If we have N users, each user gets:

Dimension per user = 2BT / N The total resource is equally split — regardless of how the split is done (time, frequency, or code)
★ Key result: TDMA, FDMA, and orthogonal CDMA all create the same number of orthogonal channels from the same signal space. In AWGN channels, all three techniques have identical capacity. The difference only appears in fading channels, where multipath and Doppler affect each technique differently.

Step 5 — Random Access (For Bursty Traffic)

Not all users transmit continuously. Sending a text message, loading a web page — these are bursty transmissions. For these, reserving a dedicated channel is wasteful. Instead, users compete for access when they need it. This is called random access (e.g., ALOHA, CSMA). Random access does not guarantee channel access — two users may transmit simultaneously causing a collision.

Practice — §14.2 Overview
  1. A system has bandwidth B = 10 MHz and symbol duration T = 100 µs. What is the total signal space dimension? If 50 users share it, how many dimensions does each get?
  2. Explain why TDMA, FDMA, and orthogonal CDMA have the same capacity in AWGN but may differ in fading channels. Give a specific example of how fading harms one more than another.
  3. Why does voice require a dedicated channel but a webpage load is better served by random access?

1. 2BT = 2 × 10×10⁶ × 100×10⁻⁶ = 2000 dimensions. Per user: 2000/50 = 40 dimensions each.

2. In AWGN all three methods orthogonally partition the same 2BT dimensions → same capacity. In fading: TDMA users experience channel variations across time slots — if the channel deep-fades during a user's slot, their entire transmission is lost. FDMA users experience channel variations across frequency — a user assigned a frequency in a spectral null performs badly. CDMA spreads across wide bandwidth, potentially averaging out frequency-selective fading. So CDMA can be more robust to frequency-selective fading.

3. Voice: constant bit rate, real-time, delay-sensitive — a 50 ms gap makes the call unintelligible. Needs guaranteed, continuous channel. Webpage: bursty, tolerates delay, needs channel only during actual data transfer (milliseconds). Reserving a full channel for idle think-time wastes spectrum. Random access assigns the channel only when data exists → much higher aggregate efficiency.

Section 14.2.1

Frequency-Division Multiple Access (FDMA)

The simplest multiple access method: divide the spectrum into slices and give each user their own slice. Like separate radio stations, all on air simultaneously on different frequencies.

Step 1 — Core Idea

In FDMA, the total bandwidth B is divided into N non-overlapping frequency channels. Each user is assigned one channel and uses it continuously.

  Total bandwidth B = 25 MHz
  ┌──────┬──────┬──────┬──────┬──────┐
  │ User1│guard │ User2│guard │ User3│  ...
  │  Bc  │      │  Bc  │      │  Bc  │
  └──────┴──────┴──────┴──────┴──────┘
  ← Bc = 30 kHz → ← Bg →← Bc = 30 kHz →
Each user occupies a separate frequency band · Guard bands separate channels

Step 2 — Guard Bands: Why They Exist

Small empty frequency gaps are placed between user channels. They are necessary because:

  • Imperfect filters — real filters don't cut off sharply; signal leaks into adjacent channels
  • Adjacent channel interference — without a gap, User 1's signal bleeds into User 2's band
  • Doppler shift — moving users experience a frequency shift: f_received = f_transmitted + f_doppler. Guard bands absorb this shift.

Step 3 — Flat Fading Per Channel

📡
FDMA naturally handles frequency-selective fading: If each FDMA channel bandwidth Bc is much smaller than the channel coherence bandwidth Bc_channel, the entire FDMA channel experiences approximately the same fading (flat fading). This simplifies the receiver — no equalizer needed for each subchannel.

Step 4 — Continuous Transmission

Once assigned a frequency channel, a user transmits continuously — like a radio station that's always on air. This has pros and cons:

Advantage

Simple for analog systems. No need for buffering or synchronisation. Channel estimation can track slowly over time.

Disadvantage

Channel estimation must occur during live data — harder to insert training sequences. Requires frequency-agile radios (must tune to exact carrier). Hard to assign multiple channels to one user (needs parallel demodulators).

Step 5 — OFDMA: FDMA in Modern Systems

Modern 4G/5G systems use OFDM subcarriers. When different subcarriers are assigned to different users, this is called OFDMA (Orthogonal Frequency Division Multiple Access):

  OFDM subcarriers:
  │sc1│sc2│sc3│sc4│sc5│sc6│sc7│sc8│sc9│...
  └───────────┘ └───────────┘ └───────┘
     User 1         User 2      User 3

  User 1 → subcarriers 1–3
  User 2 → subcarriers 4–6
  User 3 → subcarriers 7–9
OFDMA = FDMA implemented using OFDM subcarriers — used in LTE, 5G, WiFi 6

Step 6 — Number of Users Formula

N = (B − 2·Bg) / Bc B = total bandwidth · Bg = guard band at each band edge · Bc = bandwidth per user channel
Example 14.1 — Analog vs Digital FDMA (Textbook)

Given: Total bandwidth B = 25 MHz (uplink), each operator gets B = 25 MHz total.

Analog System: Bc = 30 kHz per voice channel, edge guard bands Bg = 10 kHz each side.

N = (25×10⁶ − 2×10×10³) / (30×10³) = (25,000,000 − 20,000) / 30,000 = 24,980,000 / 30,000 ≈ 832 users

Per operator: 832/2 = 416 users.

Digital System: Bc = 10 kHz (high-level modulation, 3× more efficient), Bg = 5 kHz.

N = (25×10⁶ − 2×5×10³) / (10×10³) = 24,990,000 / 10,000 = 2,499 users

Digital system supports ~3× more users! Because digital modulation can encode the same voice in 3× less bandwidth. Same spectrum, 3× more users — the power of digital communications.
Practice — §14.2.1 FDMA
  1. A 5G OFDMA system has total bandwidth B = 100 MHz, subcarrier spacing = 15 kHz, and Bg = 1 subcarrier guard on each edge. How many total subcarriers are available? If each user needs 12 subcarriers (one resource block), how many simultaneous users can be served?
  2. An FDMA system has B = 10 MHz, Bc = 25 kHz, Bg = 5 kHz. Find N. Now double the efficiency by using digital modulation (Bc = 12.5 kHz, Bg = 2.5 kHz). By how much does N increase?
  3. Why does FDMA naturally resist frequency-selective fading better than TDMA? Connect your answer to coherence bandwidth.

1. Total subcarriers = B/spacing = 100MHz/15kHz ≈ 6,667 subcarriers. Subtract 2 guard: 6,665 data subcarriers. Users = floor(6,665/12) = 555 simultaneous users.

2. N₁ = (10×10⁶ − 10×10³)/(25×10³) = 9,990,000/25,000 = 399.6 → 399 users. N₂ = (10×10⁶ − 5×10³)/(12.5×10³) = 9,995,000/12,500 = 799.6 → 799 users. Increase: 799/399 ≈ 2× more users.

3. FDMA assigns each user a narrow frequency channel of bandwidth Bc. If Bc < coherence bandwidth Bc_chan, the user experiences flat fading (same gain across their whole channel) — no ISI, no frequency-selective distortion. TDMA assigns users the full bandwidth B, which is much larger than Bc_chan → frequency-selective fading → ISI → needs equalizer. So FDMA naturally avoids ISI per user when channels are narrow enough.

Section 14.2.2

Time-Division Multiple Access (TDMA)

Instead of splitting frequency, TDMA splits time. Users take turns — rapidly cycling through time slots so fast that from the user's perspective it feels continuous.

Step 1 — Core Idea: Time Slots and Frames

In TDMA, the communication time is divided into small time slots. Multiple slots grouped together form a frame. Each user is assigned one (or more) slot per frame. The frame repeats cyclically:

  Frame (cyclically repeating):
  ┌────┬────┬────┬────┬────┬────┬────┬────┐
  │ U1 │ U2 │ U3 │ U4 │ U5 │ U6 │ U7 │ U8 │ ← Frame 1
  └────┴────┴────┴────┴────┴────┴────┴────┘
  ┌────┬────┬────┬────┬────┬────┬────┬────┐
  │ U1 │ U2 │ U3 │ U4 │ U5 │ U6 │ U7 │ U8 │ ← Frame 2
  └────┴────┴────┴────┴────┴────┴────┴────┘
  Each user transmits on their slot, then waits for next frame.
8 users share one TDMA channel — user 1 gets slot 1 in every frame

Step 2 — TDMA Uses the Full Bandwidth

Unlike FDMA (where each user has a narrow frequency band), in TDMA each user uses the entire channel bandwidth during their time slot. This means:

  • Each user's signal is wideband
  • The channel is typically frequency-selective → causes ISI
  • Therefore: ISI mitigation is usually required in TDMA systems

Step 3 — Why Digital Transmission Is Required

Users don't transmit continuously — they transmit in bursts (one slot per frame). A voice call generates data at a constant rate, but TDMA only gives a slot every frame period. So the device must:

  Voice generates: 8 kbps continuously
  TDMA frame period: 20 ms, 8 users/frame
  User gets: 1 slot every 20 ms
  
  So during one slot, must transmit: 8 kbps × 20 ms = 160 bits
  But compressed into 1/8 frame = 2.5 ms slot
  → Must transmit at 64 kbps during slot (8× faster!)
  → Need digital buffer to store 20 ms of voice, send in 2.5 ms burst
Buffering: store voice data, transmit in a burst during the allocated slot

Step 4 — Advantages of TDMA

  • Easy multi-slot allocation: A user needing 2× data rate just gets 2 slots per frame — no hardware change needed
  • Simpler channel estimation: When User 1 is transmitting, User 2 can use their idle slots to perform channel estimation, send pilot signals, or execute control signaling — without interrupting data
  • Single radio frequency: All users share the same carrier — only one oscillator/filter needed at the base station per TDMA channel

Step 5 — The Big Challenge: Synchronisation

Downlink (Base Station → Users) — Easy

All signals originate from the same transmitter. Travel through the same channel to each user. Timing is automatically preserved — if slots were orthogonal at the transmitter, they arrive orthogonal at the receiver.

Uplink (Users → Base Station) — Hard

Different users are at different distances. User A at 100m and User B at 5 km have different propagation delays. Even if they transmit at exactly the "right" time, signals arrive offset at the base station → slots overlap → lose orthogonality.

Solution: The base station measures each user's propagation delay and instructs them to transmit at a compensated time (timing advance). This coordination works but adds overhead. The base station handles all synchronisation centrally.

Step 6 — Guard Times Between Slots

Even with timing advance, small errors remain (multipath, Doppler, oscillator drift). Guard times are empty intervals between slots to absorb these:

  TDMA frame with guard times:
  │ U1 │gt│ U2 │gt│ U3 │gt│ U4 │gt│
  
  gt = guard time (empty interval)
  Absorbs: timing advance errors + multipath delay spread

Guard times reduce spectral efficiency but are essential for maintaining orthogonality.

Step 7 — Channel Re-Estimation Every Frame

Since each user only transmits once per frame, the channel may change between frames (especially for mobile users at high speed). The receiver must re-estimate the channel at each slot — it cannot track continuously. This is less efficient than FDMA where continuous transmission allows continuous tracking.

Example 14.2 — GSM System Design (Textbook)

Given: B = 25 MHz, 125 TDMA channels of 200 kHz each, 8 users per channel, delay spread Tm = 10 µs.

Total users: 125 channels × 8 slots = 1,000 users

Channel bandwidth: 25 MHz / 125 = 200 kHz per TDMA channel

Check for ISI: Coherence bandwidth Bc ≈ 1/Tm = 1/10µs = 100 kHz

Since channel bandwidth (200 kHz) > coherence bandwidth (100 kHz): frequency-selective fading → ISI present!

Conclusion: GSM requires an equalizer. GSM uses an MLSE-based equalizer (Viterbi algorithm). The type is left to the designer — the GSM standard only specifies performance requirements, not the specific equalizer implementation.

GSM Frame Structure (for reference): Each 200 kHz channel uses a 26-symbol training sequence embedded in each 148-symbol burst for channel estimation, exactly as studied in Chapter 11.

Practice — §14.2.2 TDMA
  1. A TDMA system has B = 30 MHz, 150 channels of 200 kHz each, 6 users per channel. (a) Find total users supported. (b) If delay spread Tm = 5 µs, find Bc. Does ISI occur? (c) What ISI mitigation would you recommend?
  2. User A is 500 m from the base station and User B is 3 km away. Speed of light = 3×10⁸ m/s. If they both start transmitting at time t=0, what is the difference in arrival times at the base station? If each time slot is 500 µs long, is this timing error significant?
  3. A high-speed user wants 4× the data rate of a normal TDMA user. How many slots per frame should they be assigned? What is the disadvantage of this approach?

1. (a) 150 × 6 = 900 users. (b) Bc = 1/5µs = 200 kHz. Channel BW = 200 kHz = Bc exactly → borderline frequency-selective fading → ISI present. (c) Since Bc ≈ channel bandwidth, equalization is required. An MLSE or DFE equalizer (as in Chapter 11) would be appropriate. With Bc = channel BW, the channel has mild ISI — a few-tap DFE or short MLSE would suffice.

2. User A delay: 500/(3×10⁸) ≈ 1.67 µs. User B delay: 3000/(3×10⁸) = 10 µs. Timing difference: 10 − 1.67 = 8.33 µs. Slot duration = 500 µs. The timing error is 8.33/500 ≈ 1.67% of a slot — small but non-negligible. Guard times of at least 10 µs would absorb this. The base station would instruct User A to transmit 8.33 µs later (timing advance).

3. Assign 4 slots per frame. The high-speed user gets 4× the bandwidth → 4× data rate. Disadvantage: that reduces available slots for other users (if 8 slots/frame, now only 4 remaining for 4 other users instead of 7). Also, the device must process data 4× faster in each slot — higher peak processing requirement.

Section 14.2.3

Code-Division Multiple Access (CDMA)

The most conceptually different approach: all users transmit simultaneously on the same frequency. They are separated only by their unique spreading codes. Understanding CDMA requires understanding spread spectrum.

Step 1 — Core Idea: Same Time, Same Frequency, Different Codes

  TDMA:  | User1 | User2 | User3 |  ← different TIMES
  FDMA:  |User1||User2||User3|      ← different FREQUENCIES
  CDMA:  all users transmit simultaneously at same frequency
         but each user has a unique SPREADING CODE

  At receiver:
  y(t) = User1_signal + User2_signal + User3_signal + noise
  To recover User 1: multiply y(t) by User1's code → User1 emerges,
  User2 and User3 average to near-zero (like noise)
CDMA: separation happens at receiver using codes, not at transmitter using time/frequency

Step 2 — What is a Spreading Code?

A spreading code is a high-rate binary sequence used to encode user data before transmission. Each user has a unique code. The data rate after spreading is much higher than the original data rate:

Processing Gain G = Spread bandwidth / Original signal bandwidth = Bspread / Bs G measures how much wider the spread signal is than the original signal · Typical G = 64 to 1000+
Language analogy: Many people talk simultaneously in a room, each in a different language. If you know one language, you focus on it and everything else sounds like noise. Spreading codes are like different languages — the receiver "knows" one code and can separate that user's signal.

Step 3 — Two Spread Spectrum Methods

Direct Sequence (DSSS)

Each data bit is multiplied by a high-rate chip sequence. If data bit = 1 and code = 10110101, the transmitted chip sequence is 10110101. This spreads the signal across a wide bandwidth. Used in IS-95, 3G WCDMA.

Frequency Hopping (FHSS)

The transmitter rapidly hops between frequencies according to a code sequence. Time1 → 900 MHz, Time2 → 902 MHz, etc. The receiver hops synchronously. Used in Bluetooth, military systems.

Step 4 — Orthogonal vs Non-Orthogonal Codes

Code TypeUsed WhereInterferenceWhy
Orthogonal (Walsh-Hadamard)Downlink (BS→users)Zero between usersBase station controls all transmissions → perfect timing alignment → perfect orthogonality
Non-orthogonalUplink (users→BS)Nonzero — mutual interferenceHard to synchronise many users perfectly → codes lose orthogonality → some cross-correlation remains

Advantages of Non-Orthogonal CDMA on Uplink

  • Users need minimal coordination — no strict timing requirements
  • No hard limit on number of users (can add users gracefully, accepting more interference)
  • More flexible than TDMA/FDMA where adding one more user requires a whole new channel
Trade-off: More users → more interference → lower SNR for everyone. CDMA is an interference-limited system. The number of supportable users is determined by the required SNR, not the number of orthogonal channels.

Step 5 — The Near-Far Problem

This is the most critical challenge unique to CDMA uplinks. Consider two users transmitting to the same base station:

  User A (near) ── 50 m ──► Base Station
                              ↑ receives User A very strong

  User B (far) ──── 3 km ──► Base Station
                              ↑ receives User B very weak

  Both transmit at same power:
  User A received power: 10 W (50 m away)
  User B received power: 0.01 W (3 km away, 1/r² path loss)

  User B's signal is BURIED under User A's interference!
  User A appears as strong interference to User B's recovery.
Near-far effect: strong nearby users swamp weak distant users

Step 6 — Power Control: The Solution to Near-Far

Power Control in CDMA

The base station measures the received power from each user and sends feedback: "User A, transmit less power. User B, transmit more power." The goal: all users arrive at the base station with equal received power.

Power control: each user adjusts Pt such that Pr (at base station) = constant for all users

This essentially inverts the path loss and fading — each user compensates for their channel. Result: all users appear equally strong at the base station → fair competition → near-far problem eliminated.

Challenge: power control must be fast (faster than channel fading) and accurate. In IS-95, power control updates happen 800 times per second!

Step 7 — Multiuser Detection (MUD)

Traditional CDMA receivers treat other users as noise. But MUD (Multiuser Detection) is smarter: it jointly detects all users simultaneously, explicitly accounting for their mutual interference.

  • Provides significant performance improvement even under perfect power control
  • Works even better when MUD and power control are jointly optimised
  • Optimal MUD (with non-orthogonal CDMA) achieves Shannon capacity of the uplink channel
  • More complex receiver — typically O(2^K) for K users — but reduced-complexity approximations exist

Step 8 — Easy Multi-Code Allocation

To give one user a higher data rate, simply assign them multiple codes. User 1 can have Code A, Code B, and Code C — transmitting three streams simultaneously. This is simpler than TDMA (which needs multiple slots) or FDMA (which needs multiple frequency bands).

Example 14.3 — CDMA vs FDMA Capacity (Textbook)

Given: Uplink, G = 128, bandwidth = 1.25 MHz, Bs = 1.25 MHz/128 = 9.765 kHz.

SIR formula for non-orthogonal CDMA: SIR = 3G/(K−1)

Required SIR = 10 dB = 10 (linear). Find maximum K:

K ≤ 3G/SIR + 1 = 3×128/10 + 1 = 38.4 + 1 = 39.4 → K = 39 users

FDMA comparison: 1.25 MHz / 9.765 kHz = 128 channels = 128 users

SystemUsers (theoretical)Notes
Non-orthogonal CDMA39Limited by mutual interference
FDMA128Limited by bandwidth partition
IS-95 CDMA (practical)~64Uses voice activity, variable rate, better assumptions
Key insight: Theoretical FDMA outperforms non-orthogonal CDMA in a single cell under the Gaussian assumption. BUT CDMA allows better frequency reuse across neighbouring cells (see Section 15.2), which can make the overall network capacity comparable or better. Also, IS-95 uses voice activity detection (VAF ≈ 3/8) — users aren't always talking — effectively multiplying capacity by ~2.67.
Practice — §14.2.3 CDMA
  1. A CDMA system has G = 64. What is the maximum number of users if the required SIR = 7 dB? (Use SIR = 3G/(K−1), convert 7 dB to linear first.)
  2. Explain the near-far problem with a numerical example: User A is 100 m away and User B is 2 km away, both transmitting at 1 mW. Assume path loss ∝ 1/r². What is the ratio of received powers? Why does this destroy CDMA performance?
  3. If a CDMA user needs 4× the data rate, how many codes do they need? How does this affect other users in the system?

1. 7 dB linear = 10^(7/10) = 5.012. K ≤ 3×64/5.012 + 1 = 192/5.012 + 1 = 38.3 + 1 = 39.3 → K = 39 users.

2. Path loss ∝ 1/r². Received power User A: Pt/100² = Pt/10,000. Received power User B: Pt/2000² = Pt/4,000,000. Ratio: (Pt/10,000)/(Pt/4,000,000) = 400. User A's signal is 400× stronger than User B's! At the CDMA receiver trying to decode User B: User A looks like an interference source 400× stronger than User B's signal → SIR ≈ G/400 ≈ 128/400 = 0.32 (negative dB) → complete failure. Power control must reduce User A's power by 400× (or equivalently increase User B's by 400×) to equalise received powers.

3. Need 4 codes. Each code occupies the same bandwidth and adds one more interferer for everyone else. With 4 extra codes from one high-rate user, other users' SIR = 3G/(K+3) instead of 3G/(K−1) — slightly worse. The system "looks like" 3 more users were added. If near the capacity limit, this could push other users below their minimum SIR.

Section 14.2.4

Space-Division Multiple Access (SDMA)

The fourth dimension — space. If two users are in different directions, a smart antenna can serve them simultaneously on the same frequency and time. Space becomes a reusable resource.

Step 1 — Core Idea: Direction as a New Dimension

🎯
The SDMA insight: TDMA divides time, FDMA divides frequency, CDMA divides code space. SDMA divides physical direction. If User A is to the north and User B is to the south, a directional antenna can simultaneously serve both on the same frequency band and time slot — they don't interfere because their signals are spatially separated. 
         User A (North)
              ↑
  West ←── Base ──→ East
    Station
              ↓
         User B (South)

  Beam A → focused North → serves User A
  Beam B → focused South → serves User B
  Both on same frequency, same time, no interference!
  (as long as angular separation > antenna resolution)
SDMA: directional beams serve different users in different directions simultaneously

Step 2 — When Are Spatial Channels Orthogonal?

Spatial channels are only orthogonal (no interference) if the angular separation between users exceeds the antenna's angular resolution:

Angular separation between users ≥ Angular resolution of antenna Two users at 10° and 12° may not be separable, but users at 10° and 90° are easily separated

Angular resolution improves with more antenna elements. A large antenna array can resolve much finer angular differences — but a large array requires more hardware, space, and processing power.

Step 3 — Practical SDMA: Sectorized Antennas

Building and steering narrow beams for every individual user is complex. Most real systems use sectorization — divide the 360° coverage area into N fixed sectors:

  3-Sector Base Station:
         Sector 1 (0°–120°)
              ↑
  Sector 3  ← BS → Sector 2
  (240°–360°)    (120°–240°)

  Each sector has its own directional antenna.
  High gain within sector, low interference to other sectors.
  TDMA or FDMA applied within each sector independently.
Common configurations: 3 sectors (120° each) or 6 sectors (60° each)

Within each sector, TDMA or FDMA is used. Between sectors, the same frequencies can be reused — this is the spatial reuse benefit of SDMA.

Step 4 — Mobility Challenge: Sector Handoff

When a user moves from one sector to another, the network must transfer the connection — called a sector handoff (similar to cell handoff). The system must:

  • Detect that the user has moved out of the original sector
  • Establish a connection in the new sector
  • Seamlessly transfer the ongoing call/data session

This adds signaling overhead. For dynamic beamforming (beam follows the user), the tracking must adapt as fast as the user moves.

Step 5 — SDMA in Modern Systems

TechnologySDMA FeatureHow It Works
4G LTEMU-MIMO beamformingBase station forms beams toward different users simultaneously using antenna array
5G NRMassive MIMO (64–256 antennas)Very fine angular resolution → many simultaneous spatial streams → huge capacity gain
WiFi 6 (802.11ax)MU-MIMOAccess point beams to multiple users simultaneously using 4–8 antenna arrays
All cellularSectorization (3 or 6 sectors)Physical sector antennas reduce inter-sector interference and allow frequency reuse per sector
Practice — §14.2.4 SDMA
  1. A cellular base station uses 6-sector antennas. If each sector uses FDMA with 10 frequency channels, what is the total number of simultaneous users the base station can serve? How does this compare to an omnidirectional antenna with the same 10 channels?
  2. A 5G Massive MIMO base station has 64 antennas. The angular resolution is approximately 180°/64 ≈ 2.8°. If two users are at angles 5° and 8° from the base station, can they be served simultaneously? What about users at 5° and 60°?

1. 6 sectors × 10 channels = 60 simultaneous users. Omnidirectional: only 10 channels (can't reuse frequencies — interference from all directions). SDMA (sectorization) gives 6× more users from the same spectrum! This is the core benefit of spatial reuse.

2. 5° and 8°: angular separation = 3°, resolution ≈ 2.8°. Since 3° ≈ 2.8° (barely exceeds resolution), these users are marginally separable — interference between them would be high. Not ideal, but possible with some degradation. 5° and 60°: separation = 55° ≫ 2.8° — easily and cleanly separated. The base station can form completely independent beams with near-zero inter-user interference.

Comprehensive Final Problem

Comprehensive — All Four Techniques
  1. [FDMA vs TDMA comparison] A spectrum of B = 20 MHz is to support voice users. Analog FDMA: Bc = 30 kHz, Bg = 10 kHz. Digital TDMA: 200 kHz channels, 8 users/channel, Bg = 0 (adjacent channels different freq so no guard needed). (a) Find users for each system. (b) Which supports more users? (c) If Tm = 15 µs, does TDMA need an equalizer?
  2. [CDMA capacity + near-far] CDMA uplink with G = 256, required SIR = 6 dB. (a) Find maximum users K. (b) User A is 200 m away, User B is 4 km away, both at 1 mW transmit. Find received power ratio (path loss ∝ r²). (c) By what factor must User A reduce power to equalize received power at the base station?
  3. [SDMA sectorization] A base station covers a cell using (a) omnidirectional antenna, (b) 3-sector antenna, (c) 6-sector antenna. Each sector/cell uses FDMA with 15 channels. Find total simultaneous users for each configuration. By what factor does 6-sector outperform omnidirectional?

1. (a) FDMA: N = (20×10⁶ − 2×10×10³)/(30×10³) = 19,980,000/30,000 = 666 users. TDMA: N_channels = 20×10⁶/200×10³ = 100 channels. Total users = 100×8 = 800 users. (b) TDMA supports more (800 vs 666). Digital compression + time-sharing enables more efficient spectrum use. (c) Bc = 1/15µs ≈ 67 kHz. TDMA channel BW = 200 kHz > 67 kHz → frequency-selective fading → YES, equalizer needed.

2. (a) 6 dB linear = 10^0.6 = 3.981. K ≤ 3×256/3.981 + 1 = 768/3.981 + 1 = 192.9 + 1 = 193.9 → K = 193 users. (b) Path loss ∝ r². User A: Pr_A = 1 mW/200² = 1/(40,000) mW. User B: Pr_B = 1 mW/4000² = 1/(16,000,000) mW. Ratio: Pr_A/Pr_B = 16,000,000/40,000 = 400. User A is 400× stronger. (c) User A must reduce power by factor 400. If User B is at 1 mW, User A must transmit at 1/400 mW = 2.5 µW to equalize.

3. (a) Omnidirectional: 15 users. (b) 3-sector: each sector uses same 15 channels, 3 sectors → 3×15 = 45 users. (c) 6-sector: 6×15 = 90 users. 6-sector outperforms omnidirectional by factor 6×! This is pure spatial reuse — same spectrum used 6 times.
